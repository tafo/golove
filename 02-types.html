📥 Veri tutmak Holding data

📝🔢💰✅

isim · yaş · fiyat · aktif mi name · age · price · active?

Programlar veri tutar. Veri tutmak için değişken lazım. Programs hold data. To hold data, you need variables.

Her değişkenin bir tipi vardır: yazı mı, sayı mı, doğru/yanlış mı? Tip Go'ya ne tutacağını söyler. Every variable has a type: text, number, true/false? The type tells Go what to expect.

Nasıl tanımlarız? How do we declare one?

2/12

📝 var ile tanımla Declare with var 

var isim string = "Ada"
var yas int = 12

var [isim] [tip] = [değer] — resmi yol. var [name] [type] = [value] — the formal way.

Üç parça: isim (ne dediğin) · tip (ne tutacağı) · değer (başlangıç). Three parts: name (what you call it) · type (what it holds) · value (starting point).

Daha kısa bir yol var… There's a shorter way…

3/12

⚡ := kısa yol := short way 

isim := "Ada"
yas := 12

// Tip otomatik anlaşılır
// isim → string · yas → int

:= aynı işi daha kısa yapar. := does the same thing, shorter.

Tipi sen yazmazsın — Go değerden anlar. Function içinde en sık yazacağın yol budur. You don't write the type — Go infers it from the value. This is what you'll write most inside functions.

Hangisini ne zaman? When to use which?

4/12

🆚 var vs := var vs :=

isim := "Ada"→ function içinde, en kısa
var isim = "Ada"→ paket seviyesinde, tip otomatik
var isim string→ değer yok, sıfır değerle başla
isim := "Ada"→ inside a function, shortest
var isim = "Ada"→ at package level, type inferred
var isim string→ no value yet, starts at zero

Function içinde :=. Üst seviyede var. Inside a function: :=. At top level: var.

:= sadece function içinde çalışır. Paket seviyesinde mecburen var. := only works inside functions. At package level, you must use var.

Hangi tipler var? What types exist?

5/12

🎨 Temel tipler Basic types 

var isim   string  = "Ada"      // metin
var yas    int     = 12         // tam sayı
var fiyat  float64 = 9.99       // ondalık
var aktif  bool    = true       // doğru/yanlış

string · int · float64 · bool — gündelik kullanımda %95. string · int · float64 · bool — these cover 95% of daily use.

Daha sonra göreceğin: int8/16/32/64, uint, byte, rune. Şimdilik bu dördü yeter. You'll meet later: int8/16/32/64, uint, byte, rune. For now these four are plenty.

Değer vermezsen ne olur? What if you don't give a value?

6/12

0️⃣ Sıfır değerler Zero values 

var s string     // → ""    (boş string)
var n int        // → 0
var f float64    // → 0.0
var b bool       // → false

Değer vermezsen Go bir sıfır değer atar. If you give no value, Go assigns a zero value.

Go'da hiçbir zaman undefined ya da null bombası yok. Her değişken her zaman geçerli bir değere sahiptir. No undefined or null surprises in Go. Every variable always has a valid value.

Tipler arası geçiş? Converting between types?

7/12

🔄 Tip dönüşümü Type conversion 

yas := 12
fiyat := float64(yas)   // int → float64

ondalik := 9.7
sayi := int(ondalik)     // → 9 (kesilir)

Tipler arası geçiş açıkça yazılır: T(x). Type conversion is explicit: T(x).

Go otomatik dönüştürmez. int + float64 bile derleme hatası — birini diğerine çevirmek senin sorumluluğun. Go won't auto-convert. Even int + float64 is a compile error — converting is your job.

Değişmeyecek değerler? Values that never change?

8/12

🔒 const — sabit değer const — fixed value 

const PI = 3.14159
const ULKE = "Türkiye"

PI = 3.14      // ✗ derleme hatası

Değişmeyecek değerler için const. For values that won't change, use const.

Compile zamanında bilinen sabitler. Tekrar atamaya çalışırsan derleme hatası verir. Constants known at compile time. Try to reassign and you get a compile error.

Birden çoğunu birden tanımlamak? Declaring many at once?

9/12

📚 Çoklu tanımlama Multiple declarations 

var (
    isim  string = "Ada"
    yas   int    = 12
    aktif bool   = true
)

x, y := 10, 20            // tek satırda iki değişken

var (...) blok · a, b := 1, 2 tek satır. var (...) block · a, b := 1, 2 one line.

İlişkili değişkenleri gruplamak okunaklılığı artırır. Function dönüş değerlerinde de göreceğiz: x, err := f(). Grouping related variables improves readability. You'll see this with function returns too: x, err := f().

İsimler nasıl yazılır? How are names written?

10/12

🔤 Adlandırma kuralı Naming rule 

var isim string     // küçük harf → paket içi
var Isim string     // büyük harf → dışa açık

const userKey = ...   // paket içi
const UserKey = ...   // dışa açık

Büyük harfle başlar → dışa açık. Küçük → paket içi. Starts capital → exported. Lowercase → package-private.

Go'nun en şık özelliği: public/private keyword yok. İlk harfin büyüklüğü her şeyi söyler. Go's slickest feature: no public/private keyword. The first letter's case says it all.

Bir değişkenin tipini görmek? How to see a variable's type?

11/12

🔍 %T ile tipi gör See the type with %T 

yas := 42
fmt.Printf("%T\n", yas)     // int

fiyat := 9.99
fmt.Printf("%T\n", fiyat)   // float64

isim := "Ada"
fmt.Printf("%T\n", isim)    // string

%T bir değerin tipini yazar. %T prints a value's type.

Bir değişkenin ne tuttuğundan emin olmadığında debug için altın değerinde. Gold for debugging when you're unsure what type a variable holds.

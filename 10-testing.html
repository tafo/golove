🧪 Niye test? Why test? 

// Manuel test: her seferinde elle dene.
// Otomatik test: bir kez yaz, milyon kez koştur.
//
// Hangi kodun bozuk olduğunu bilmek
// hangi kodun bozuk olduğunu sanmaktan iyidir.

Test = "kod hâlâ söz verdiği şeyi yapıyor mu?" sorusu. Tests = "is the code still doing what it promised?".

Go'nun stdlib'i test çalıştırır. Üçüncü parti kütüphane yok, framework yok, kurulum yok. Go's stdlib runs tests. No third-party library, no framework, no setup.

Test dosyası nasıl? What does a test file look like?

2/11

📁 _test.go ile bitir End the file with _test.go 

math.go         // asıl kod
math_test.go    // testler

// Aynı paket. Aynı klasör. Yan yana.

Test dosyalarının adı _test.go ile biter. Test files end with _test.go.

Go derleyicisi _test.go dosyalarını binary'ye dahil etmez. Sadece go test çalıştığında derlenir. Go's compiler skips _test.go in builds. They're only compiled when you run go test.

Test fonksiyonunun şekli? The shape of a test function?

3/11

🧩 İlk test fonksiyonu First test function 

package math

import "testing"

func TestTopla(t *testing.T) {
    sonuc := Topla(2, 3)
    if sonuc != 5 {
        t.Errorf("2+3 = %d, beklenen 5", sonuc)
    }
}

func TestX(t *testing.T) — sabit bir kalıp. func TestX(t *testing.T) — fixed pattern.

İsim mutlaka Test ile başlamalı (büyük T). Tek argüman: *testing.T. Dönüş yok. Name must start with Test (capital T). One arg: *testing.T. No return.

Hata bildirmek? Reporting failure?

4/11

📛 t.Errorf vs t.Fatalf t.Errorf vs t.Fatalf 

t.Errorf("yanlış: %d", x)   // hata yaz, devam et
t.Fatalf("yanlış: %d", x)   // hata yaz, dur

t.Helper()                  // "ben yardımcıyım, beni atla"

Errorf = "fail ama devam". Fatalf = "fail ve dur". Errorf = "fail but continue". Fatalf = "fail and stop".

Devam edersen birden fazla hatayı tek koşuda görebilirsin. Bir adım sonraki kontrolün anlamsız olduğu durumlarda Fatalf. Continuing lets you see many failures in one run. Use Fatalf when subsequent checks are meaningless.

Çalıştırmak? Running them?

5/11

▶️ go test komutları go test commands 

$ go test                  # bu paketteki tüm testler
$ go test ./...            # tüm modüldeki tüm testler
$ go test -v               # detaylı çıktı
$ go test -run TestTopla   # sadece adı eşleşen
$ go test -count=1         # cache'i kullanma

Tek komut, çok seçenek. go test ./... en sık. One command, many flags. go test ./... is the everyday one.

Go test sonuçlarını cache'ler — hiçbir şey değişmediyse tekrar çalıştırmaz. -count=1 bunu kapatır. Go caches test results — won't re-run if nothing changed. -count=1 bypasses the cache.

Aynı testin alt vakaları? Sub-cases of one test?

6/11

🌳 t.Run — alt testler t.Run — sub-tests 

func TestTopla(t *testing.T) {
    t.Run("pozitifler", func(t *testing.T) {
        if Topla(2, 3) != 5 { t.Fail() }
    })
    t.Run("sıfırlar", func(t *testing.T) {
        if Topla(0, 0) != 0 { t.Fail() }
    })
}

İlişkili kontrolleri tek test fonksiyonu altında grupla. Group related checks under one test function.

Alt testleri ayrı ayrı çalıştırabilirsin: go test -run TestTopla/pozitifler. Run sub-tests individually: go test -run TestTopla/pozitifler.

Aynı testi farklı verilerle? Same test, many inputs?

7/11

📋 Tablo testleri Table-driven tests 

vakalar := []struct {
    ad   string
    a, b int
    sonuc int
}{
    {"pozitif", 2, 3, 5},
    {"sıfır",   0, 0, 0},
    {"negatif", -1, 1, 0},
}

for _, v := range vakalar {
    t.Run(v.ad, func(t *testing.T) {
        if Topla(v.a, v.b) != v.sonuc { t.Fail() }
    })
}

Vakalar bir slice'ta. Tek döngü, çok test. Go'nun favori kalıbı. Cases in a slice. One loop, many tests. Go's favourite pattern.

Yeni vaka eklemek = bir satır slice'a ekle. Test eklemek bu kadar ucuzsa daha çok yazarsın. Adding a new case = one line in the slice. When tests are this cheap, you write more.

Test örnekleri belge olarak? Tests as documentation?

8/11

📖 Example — çalışan örnek Example — runnable example 

func ExampleTopla() {
    fmt.Println(Topla(2, 3))
    // Output: 5
}

Hem dokümantasyon, hem test. Documentation and test in one.

go doc bunu kullanım örneği olarak gösterir, go test da çalıştırıp // Output: ile karşılaştırır. go doc shows it as a usage example, and go test runs it and compares to // Output:.

Hangi satırlar test edildi? Which lines got tested?

9/11

📊 Code coverage Code coverage 

$ go test -cover
PASS
coverage: 87.5% of statements

$ go test -coverprofile=cov.out
$ go tool cover -html=cov.out
# tarayıcıda hangi satır test edildi/edilmedi gör

-cover hangi satırların test edildiğini ölçer. -cover measures which lines got hit by tests.

%100 hedef değil — anlamlı testler yaz. Ama coverage düştüğünde "buraya kim baktı?" sormak değerli. 100% isn't the goal — write meaningful tests. But a drop in coverage is a useful "who looked here?" prompt.

Hızı ölçmek? Measuring speed?

10/11

⏱️ Benchmark Benchmark 

func BenchmarkTopla(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Topla(2, 3)
    }
}

// $ go test -bench=.
// BenchmarkTopla-8  1000000000  0.5 ns/op

Benchmark = "bu kod ne kadar hızlı?" testi. Benchmark = "how fast is this?" test.

b.N Go tarafından ayarlanır — yeterince istikrarlı bir ölçüm için döngü tekrarı. Go sets b.N — the loop count is chosen to give a stable measurement.

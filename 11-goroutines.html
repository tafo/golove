🚀 go keyword'u The go keyword 

go selamla("Ada")
go selamla("Mehmet")

// İki function aynı anda çalışmaya başlar.
// Hangisi önce biter? Belli değil.

Bir çağrının önüne go yaz — hafif bir thread'de çalışsın. Put go in front of a call — it runs in a lightweight thread.

Buna goroutine denir. Tek anahtar kelime, tek satır. Eşzamanlılığı bu kadar kolay yapan az dil var. It's called a goroutine. One keyword, one line. Few languages make concurrency this cheap.

💀 main biterse hepsi biter When main exits, all die 

func main() {
    go fmt.Println("merhaba")
    // main hemen biter
    // goroutine "merhaba" yazma şansı bulamayabilir
}

main goroutine bittiğinde program kapanır — diğerleri yarım kalır. When the main goroutine ends, the program exits — others get cut off.

Yeni başlayanların 1. tuzağı. "Bekle" demek için sync.WaitGroup ya da channel kullanırsın. Beginner trap #1. To "wait", use sync.WaitGroup or a channel.

Hepsi bitsin diye beklemek? Waiting for them all?

4/10

🔢 sync.WaitGroup sync.WaitGroup 

var wg sync.WaitGroup

for _, ad := range []string{"Ada", "Mehmet"} {
    wg.Add(1)
    go func() {
        defer wg.Done()
        selamla(ad)
    }()
}

wg.Wait()    // hepsi bitene kadar bekle

Sayaç gibi: Add arttırır, Done azaltır, Wait sıfır olana kadar bekler. Like a counter: Add increases, Done decreases, Wait blocks until zero.

Klasik kalıp: go'dan önce Add(1), içeride defer wg.Done(). Classic pattern: Add(1) before go, defer wg.Done() inside.

🪶 Goroutine ucuzdur Goroutines are cheap 

// 10 binlerce, hatta milyon goroutine sorun değil.
//
// Bir thread:    ~1 MB bellek, OS yönetimi
// Bir goroutine: ~2 KB başlangıç, Go runtime yönetimi
//
// Yani: 1 thread = ~500 goroutine

Thread değil. Çok daha hafif. Korkmadan çoğalt. Not threads. Much lighter. Spawn freely.

Go runtime'ı goroutine'leri thread'lere dağıtır. Sen sadece "iş başlasın" dersin, gerisi otomatik. Go's runtime maps goroutines onto OS threads. You just say "start it" — Go handles the rest.

Aynı veriye iki goroutine? Two goroutines, same data?

6/10

⚠️ Race condition Race condition 

sayac := 0

for i := 0; i < 1000; i++ {
    go func() { sayac++ }()
}

// "1000" beklersin, ama 873, 921, 998... rastgele
// 💥 İki goroutine aynı anda yazıyor — bozulur

İki goroutine'ı aynı veriye yaz/oku korumasız bırakma. Don't let two goroutines read/write the same data unprotected.

Çözüm: channel (Bölüm 12) ya da mutex (Bölüm 13). Go felsefesi: "veri paylaşma, mesaj yolla". Solution: channel (Ch. 12) or mutex (Ch. 13). Go's motto: "Don't share memory; communicate."

Yarışı nasıl yakalarsın? How do you catch a race?

🎯 go run -race go run -race 

$ go run -race main.go

// Race detector veri yarışını tespit eder.
// Hangi goroutine, hangi satır — hepsini söyler.

-race bayrağı yarış durumlarını yakalar. The -race flag catches race conditions.

Programı yavaşlatır (~10×). Geliştirmede ve CI'da çalıştır, prod'da değil. It slows the program (~10×). Use it in dev and CI, not in production.

8/10

🪤 Loop değişkeni tuzağı Loop variable trap 

// Go 1.22 öncesi:
for _, ad := range isimler {
    go func() {
        fmt.Println(ad)   // 💥 hep son ismi yazar
    }()
}

// Çözüm — değeri içeride yakala:
for _, ad := range isimler {
    ad := ad
    go func() { fmt.Println(ad) }()
}

Eski Go'da loop değişkeni paylaşılırdı. Go 1.22+ artık her iterasyonda yeniler. In old Go, loop vars were shared. Go 1.22+ creates a new one each iteration.

1.22+ kullanıyorsan rahat ol. Ama eski koda bakacaksan bu kalıbı çok göreceksin. If you're on 1.22+, breathe easy. Reading older code? You'll see this pattern often.

⏹️ context — iptal sinyali context — cancel signal 

ctx, cancel := context.WithTimeout(
    context.Background(), 2*time.Second,
)
defer cancel()

go isYap(ctx)

// isYap içinde: ctx.Done() kanalını dinle, gelirse bitir.

context = "artık iptal" demenin standart yolu. context = standard way to say "cancel now".

Network, DB, uzun süren her iş context.Context alır. Kullanıcı kapatırsa, timeout dolarsa — temiz iptal. Network, DB, any long task takes a context.Context. User cancels or timeout fires → clean shutdown.

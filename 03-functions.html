🧰 Niye fonksiyon? Why functions?

📦📦📦

aynı işi tekrar yazma — paketle, isim ver, çağır don't repeat code — package it, name it, call it

Fonksiyon = isimlendirilmiş bir iş parçası. A function = a named piece of work.

Aynı kodu farklı yerlerde tekrar tekrar yazmazsın. Bir kez yazarsın, ihtiyaç oldukça çağırırsın. You don't write the same code in many places. Write it once, call it whenever.

Nasıl yazılır? How do we write one?

2/11

📐 İlk fonksiyon First function 

func selamla() {
    fmt.Println("Merhaba!")
}

selamla()    // → Merhaba!

func · isim · () · gövde. func · name · () · body.

Tanımlamak başka, çağırmak başka. Tanımladıktan sonra selamla() ile çalıştırırsın. Defining is one thing, calling another. Once defined, run it with selamla().

Fonksiyona veri vermek? Passing data in?

3/11

📥 Parametre — input al Parameters — taking input 

func selamla(ad string) {
    fmt.Println("Merhaba,", ad)
}

selamla("Ada")     // → Merhaba, Ada
selamla("Mehmet")  // → Merhaba, Mehmet

Parantez içine isim tip yaz. Inside parens, write name type.

Aynı fonksiyon, farklı argümanla farklı sonuç. Tekrar kullanmanın özü budur. Same function, different argument, different result. That's the essence of reuse.

Fonksiyondan değer almak? Getting a value back?

4/11

📤 Return — output ver Return — give output 

func topla(a int, b int) int {
    return a + b
}

sonuc := topla(3, 5)   // 8

Parantezden sonra dönüş tipi. Gövdede return. After parens: return type. In the body: return.

Sıra: func ad(parametreler) dönüşTipi { ... }. C/Java'nın aksine tip sonra yazılır. Order: func name(params) returnType { ... }. Unlike C/Java, the type comes after.

Aynı tipte iki parametre? Daha kısa yol var. Two params of the same type? There's a shortcut.

5/11

🪜 Aynı tipte parametreler Same-type params 

// Uzun yazım
func topla(a int, b int) int { ... }

// Kısa yazım — aynı tipi tek seferde
func topla(a, b int) int { ... }

Yan yana aynı tipliler tek tipi paylaşır. Adjacent params of the same type share one annotation.

Üç parametre aynıysa: (a, b, c int). Karışık olduğunda hepsini ayrı yaz. If three are the same: (a, b, c int). When mixed, write each separately.

Birden çok şey döndürmek? Returning more than one thing?

6/11

🎯 Çoklu dönüş değeri Multiple return values 

func bol(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("sıfıra bölme")
    }
    return a / b, nil
}

sonuc, err := bol(10, 2)   // sonuc=5, err=nil

Go'nun imzası: (sonuç, error). Go's signature move: (result, error).

Diğer dillerde exception fırlatılır. Go'da hata bir dönüş değeri olarak gelir. Kontrol etmek senin sorumluluğun. Other languages throw exceptions. Go returns errors as values. Checking them is your job.

Bir dönüşü göz ardı etmek? Ignoring a return?

7/11

🧹 _ ile boşa at Discard with _ 

// err'ı umursamıyorsan:
sonuc, _ := bol(10, 2)

// sadece err'a bakacaksan:
_, err := bol(10, 0)

_ = "bunu istemiyorum, çöpe at". _ = "I don't want this one, drop it".

Go kullanılmayan değişkene izin vermez. _ resmen "kullanılmadı sayma" demek. Go won't allow unused variables. _ officially means "don't count this as unused".

Sayısı belirsiz argüman? Variable number of arguments?

8/11

✨ Variadic — ... ile çoklu argüman Variadic — ... for many args 

func topla(sayilar ...int) int {
    toplam := 0
    for _, n := range sayilar {
        toplam += n
    }
    return toplam
}

topla(1, 2, 3)         // 6
topla(10, 20, 30, 40)  // 100

...tip = "kaç tane geleceği belli değil". ...type = "any number of these".

Function içinde slice gibi davranır. fmt.Println da bunu kullanır — bu yüzden istediğin kadar argüman verebilirsin. Inside the function it acts like a slice. fmt.Println uses this — that's why you can pass any number of args.

Function bittiğinde bir şey çalıştırmak? Run something when function ends?

9/11

⏰ defer — sonda çalıştır defer — run at the end 

func main() {
    defer fmt.Println("3. sonra")
    defer fmt.Println("2. ondan önce")
    fmt.Println("1. önce")
}
// → 1. önce
//   2. ondan önce
//   3. sonra

defer bir işi function'ın sonuna ertelemek için. defer postpones work to the end of the function.

Sıra ters: son yazılan ilk çalışır (LIFO). Klasik kullanım: defer file.Close() — açtığını mutlaka kapat. Order is reversed (LIFO). Classic use: defer file.Close() — always close what you opened.

Function bir değer mi? Is a function a value?

10/11

🌀 Fonksiyon = değer A function is a value 

selam := func(ad string) string {
    return "Merhaba, " + ad
}

fmt.Println(selam("Ada"))   // Merhaba, Ada

Fonksiyon değişkene atanabilir. Argüman olarak da verilebilir. Functions can be assigned to variables. They can be passed around.

slices.SortFunc, goroutine'ler, callback'ler — Go'nun her yerinde göreceğin bir kalıp. slices.SortFunc, goroutines, callbacks — a pattern you'll see everywhere in Go.

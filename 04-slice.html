📋 Liste lazım You need a list

🍞🥛🥚🧀🍅 …ve devamı …and more

1 ürün → 5 ürün → 100 ürün 1 item → 5 items → 100 items

Tek şey için değişken. Çok şey için liste. One thing? Use a variable. Many things? Use a list.

Go'da bu listeye slice diyoruz. Sayı değişiyorsa slice kullan. In Go, this list is called a slice. If the count can change, use a slice.

Slice nasıl yazılır? How do you write a slice?

2/11

📦 Slice böyle yazılır This is how you write a slice 

sehirler := []string{"İstanbul", "Ankara", "İzmir"}

Köşeli parantez. Tip. Süslü parantez. Elemanlar. Square brackets. Type. Curly braces. Elements.

[]string = string listesi.
:= = yeni değişken yarat. []string = a list of strings.
:= = make a new variable.

Sonradan eleman eklemek? Adding more later?

3/11

➕ append ile büyüt Grow it with append 

sehirler = append(sehirler, "Bursa")

// [İstanbul Ankara İzmir Bursa]

append yeni eleman ekler. Slice büyür. append adds a new element. The slice grows.

İlk argüman: hangi slice.
Sonraki: ne ekliyorsun. Bir veya daha fazla. First argument: which slice.
Then: what to add. One or more.

Ama küçük bir tuzak var… But there's a small trap…

4/11

⚠️ append'in tuzağı The append trap 

sehirler = append(sehirler, "Bursa")   // ✓

append(sehirler, "Bursa")              // ✗

Sonucu geri ata. sehirler = kısmını yazmayı unutma. Assign the result back. Don't forget the sehirler = part.

append yeni bir liste döner. Yakalamazsan kaybolur. append returns a new list. If you don't catch it, it's lost.

Eklediklerine nasıl bakarsın? How do you read what you added?

5/11

🔢 İndex ile eriş Read with an index 

sehirler[0]                    // "İstanbul"
sehirler[1]                    // "Ankara"
sehirler[len(sehirler)-1]       // son eleman / last

0'dan başlar. Son eleman: len(slice)-1. Starts at 0. Last element: len(slice)-1.

len() kaç eleman olduğunu söyler.
Negatif index yok. s[-1] Go'da hata verir. len() tells you how many elements there are.
No negative indexing. s[-1] is an error in Go.

Ya yanlış index verirsen? What if the index is wrong?

6/11

💥 Panic — program durur Panic — the program stops 

sehirler := []string{"İstanbul", "Ankara"}
sehirler[5]

// 💥 panic: runtime error:
//    index out of range [5] with length 2

Olmayan indeks → program durur. Wrong index → program stops.

Buna panic denir. Go hata mesajı verir. Hangi satırda olduğunu söyler. This is called a panic. Go shows an error message. It tells you the line.

Slice'tan parça almak? Taking a slice of a slice?

7/11

✂️ Parça al Take a piece 

sehirler := []string{"İst", "Ank", "İzm", "Brs"}
ortalar := sehirler[1:3]

// ortalar: ["Ank", "İzm"]

slice[başlangıç:bitiş]. Bitiş hariç. slice[start:end]. End is excluded.

[1:3] → 1 ve 2'yi al. 3 yok.
Yani 2 eleman dönüyor. [1:3] → take 1 and 2. Skip 3.
So you get 2 elements.

Ama parça aldığında bir sürpriz var… But there's a surprise when you take a piece…

8/11

🪞 Parça pencere açar A piece is a window, not a copy 

sehirler := []string{"A", "B", "C"}
parca := sehirler[0:2]
parca[0] = "X"

fmt.Println(sehirler)
// [X B C]  😱

Parça yeni liste değil. Aynı verinin penceresi. A piece is not a new list. It's a window to the same data.

Parçayı değiştirdin mi, asıl liste de değişir.
Bağımsız kopya istiyorsan: bir sonraki kart. Change the piece, the original changes too.
Want an independent copy? Next card.

Gerçek kopya nasıl? How do you make a real copy?

9/11

📋 Gerçek kopya: Clone A real copy: Clone 

import "slices"

kopya := slices.Clone(sehirler)
kopya[0] = "X"

// sehirler değişmez / unchanged

slices.Clone bağımsız bir kopya verir. slices.Clone gives an independent copy.

Go 1.21 (2023) ile geldi. Kısa ve net.
Asıl liste korunmalıysa bunu kullan. Added in Go 1.21 (2023). Short and clear.
Use it when the original must stay the same.

slices paketinde başka neler var? What else is in the slices package?

10/11

🛠️ Hazır araçlar Built-in tools

slices.Contains(s, x) // var mı? slices.Sort(s) // sırala slices.Reverse(s) // ters çevir slices.Index(s, x) // kaçıncı? slices.Equal(a, b) // aynı mı?
slices.Contains(s, x) // is it there? slices.Sort(s) // sort slices.Reverse(s) // reverse slices.Index(s, x) // at what index? slices.Equal(a, b) // same?

Döngü yazmadan önce slices paketine bak. Before writing a loop, check the slices package.

Go ekibi yazdı. Test edildi. Hızlı. Sen de kullan. Written by the Go team. Tested. Fast. Just use it.

1/11

🏗️ Niye struct? Why a struct? 

// Bu çirkin:
ad1 := "Ada";    yas1 := 12
ad2 := "Mehmet"; yas2 := 25

// Struct ile temiz:
type Kisi struct {
    Ad  string
    Yas int
}

İlişkili verileri tek bir paket olarak tutar. A struct keeps related data in one package.

Bir kişi ad ve yaştan oluşur — ayrı tutmak akılda kalmaz. Struct ikisini birleştirir. A person has a name and an age — separating them is messy. A struct ties them together.

Nasıl tanımlarız? How do we define one?

2/11

📋 Struct tanımla Define a struct 

type Kitap struct {
    Baslik  string
    Yazar   string
    Yil     int
}

type [İsim] struct { alanlar } type [Name] struct { fields }

Tip adı ve alanlar büyük harfle başlarsa dışa açık olur. Küçük olursa paket içi kalır. Capitalised type or field name → exported. Lowercase → package-private.

Değer nasıl yaratılır? How to create a value?

3/11

🆕 Değer yarat Create a value 

// 1. Alan adlarıyla — okunaklı
k := Kitap{
    Baslik: "Tutunamayanlar",
    Yazar:  "Oğuz Atay",
    Yil:    1971,
}

// 2. Sırayla — kısa ama riskli
k := Kitap{"Tutunamayanlar", "Oğuz Atay", 1971}

// 3. Boş — sıfır değerlerle
var k Kitap     // {"" "" 0}

Üç yol — alan adlarıyla yazmak en güvenlisi. Three ways — naming the fields is safest.

Sırayla yazımda alan eklersen tüm kullanımlar kırılır. Alan adlarıyla yazsan kırılmaz. Positional form breaks every usage when you add a field. Named form survives.

Alanlara erişmek? Reading the fields?

4/11

🔍 Alanlara eriş — nokta Field access — dot 

fmt.Println(k.Baslik)   // Tutunamayanlar

k.Yil = 1972            // güncelle
k.Baslik = "Tehlikeli Oyunlar"

değer.alan ile oku ve yaz. Use value.field to read and write.

Hiçbir özel sözdizimi yok. Nokta operatörü tek başına yeterli — getter/setter yazma. No special syntax. The dot operator alone does it — no need to write getters/setters.

Yapıya davranış eklemek? Adding behaviour?

5/11

🛠️ Metot — yapıya bağlı fonksiyon Method — function tied to a type 

func (k Kitap) Etiket() string {
    return k.Baslik + " — " + k.Yazar
}

k.Etiket()   // "Tutunamayanlar — Oğuz Atay"

func (alıcı Tip) İsim() ... — fonksiyon yapıya bağlanır. func (receiver Type) Name() ... — function attached to a type.

Parantezdeki alıcı (receiver) = "bu metot hangi tipe ait?" Çağırırken k.Etiket() şeklinde nokta ile. The receiver in parens says "which type owns this method?" Call it with a dot: k.Etiket().

Yapıyı değiştirmek istiyorsan? What if you need to mutate?

6/11

📌 Pointer receiver — değiştir Pointer receiver — to mutate 

// Kopya alır, değişiklik dışarı yansımaz:
func (k Kitap) YilArtir()  { k.Yil++ }

// Pointer alır, asıl yapıyı değiştirir:
func (k *Kitap) YilArtir() { k.Yil++ }

Değiştirmek istiyorsan *Tip ile alıcı al. To mutate, use a pointer receiver: *Type.

Genel kural: bir metot grubunda tutarlı kal — ya hepsi pointer, ya hepsi değer. Karışım kafa karıştırır. Rule of thumb: stay consistent within a method set — all pointer, or all value. Mixing confuses readers.

Değer yaratmak için kalıp? A pattern for creating values?

7/11

🏭 NewX — yapıcı kalıbı NewX — the constructor pattern 

func NewKitap(baslik, yazar string) Kitap {
    return Kitap{
        Baslik: baslik,
        Yazar:  yazar,
        Yil:    2026,    // varsayılan
    }
}

k := NewKitap("Saatleri Ayarlama", "Tanpınar")

Go'da constructor keyword'u yok — sadece NewX isimli fonksiyon yazılır. Go has no constructor keyword — write a function called NewX.

Varsayılan değerler veya doğrulama gerekiyorsa kullan. Basit yapılarda direkt literal de yeterli. Use it when you need defaults or validation. For simple structs, the literal is enough.

İki struct aynı mı? Are two structs equal?

8/11

⚖️ == ile karşılaştır Compare with == 

a := Kitap{"X", "Y", 2020}
b := Kitap{"X", "Y", 2020}

a == b    // true (her alan aynı)

// ⚠ İçinde slice/map varsa == hata verir!

Tüm alanlar karşılaştırılabilirse == çalışır. == works if all fields are comparable.

Slice, map, function alanı varsa == derleme hatası verir. reflect.DeepEqual daha hoşgörülü. If a struct contains slice, map, or function fields, == won't compile. reflect.DeepEqual is more forgiving.

Bir tipi başka tipe gömmek? Embedding one type in another?

9/11

↔️ Embedding — kompozisyon Embedding — composition 

type Adres struct { Sehir string }

type Kullanici struct {
    Ad string
    Adres     // alan adı yok — gömülü
}

u := Kullanici{"Ada", Adres{"İzmir"}}
fmt.Println(u.Sehir)   // "İzmir" — direkt!

Tip adını alan ismi olmadan yaz → alanları "yukarı çıkar". Write the type without a field name → its fields "promote up".

Inheritance değil, kompozisyon. Go'da "is-a" yerine "has-a" kalıbı tercih edilir. Not inheritance — composition. Go prefers "has-a" over "is-a".

Sıfır değer ne demek struct'ta? What's the zero value of a struct?

10/11

0️⃣ Sıfır değerli struct Zero-value struct 

var k Kitap

// k.Baslik = ""
// k.Yazar  = ""
// k.Yil    = 0

// Yine de geçerli bir Kitap. Hata atmaz.

Değer atamadan da kullanılabilir bir struct olabilir. A struct can be useful even with no values assigned.

İyi tasarlanmış Go yapıları sıfır değerleriyle doğrudan kullanılabilir. bytes.Buffer, sync.Mutex böyledir. Well-designed Go structs are useful from their zero value. bytes.Buffer, sync.Mutex are like this.

11/11

🎯 Özet Summary

veri + davranış = struct + metot

data + behaviour = struct + methods

Tanımla → yarat → eriş → metot ekle. Define → create → access → add methods.

📋 tanımla: type X struct { ... }
🆕 yarat: X{Alan: değer}
🔍 eriş: x.Alan
🛠️ metot: func (x X) Ad() {}
📌 değiştir: pointer receiver *X
🏭 yapıcı: NewX(...) kalıbı
↔️ embed: alan adı olmadan tip 📋 define: type X struct { ... }
🆕 create: X{Field: value}
🔍 access: x.Field
🛠️ method: func (x X) Name() {}
📌 mutate: pointer receiver *X
🏭 constructor: NewX(...) pattern
↔️ embed: a type without a field name

🌐 Niye Go ile HTTP? Why HTTP in Go? 

// Diğer dillerde: Express, FastAPI, Spring Boot...
// Go'da: stdlib içinde "net/http"
//
// Kütüphane yok. Framework yok.
// Tek import. Hazır production sunucu.

Go'nun standart kütüphanesi tek başına HTTP sunucu yazar. Go's standard library is enough to write an HTTP server.

Production'da çoğu Go API'si üçüncü parti framework olmadan yazılır. net/http hızlıdır, küçüktür, dengelidir. Most production Go APIs ship without a framework. net/http is fast, small, and stable.

İlk sunucu? A first server?

2/11

🚀 Hello, World sunucusu Hello, World server 

package main

import (
    "fmt"
    "net/http"
)

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "Merhaba, dünya!")
    })
    http.ListenAndServe(":8080", nil)
}

8 satır. Tam çalışan bir HTTP sunucu. 8 lines. A working HTTP server.

go run main.go sonra localhost:8080'i aç. İşte bu kadar. go run main.go, then open localhost:8080. That's it.

Handler nedir? What's a handler?

3/11

🤝 Handler imzası The handler signature 

func handler(w http.ResponseWriter, r *http.Request) {
    // w → cevabı yazdığın yer
    // r → gelen istek (yol, method, body, header...)
}

Her handler iki argüman alır: w cevap, r istek. Every handler takes two args: w for response, r for request.

w bir io.Writer — fmt.Fprintf, json.NewEncoder gibi her şey çalışır. r isteğin tamamını taşır. w is an io.Writer — fmt.Fprintf, json.NewEncoder, etc. all work. r carries the full request.

Cevap döndürmek? Sending a response?

6/11

📤 Cevap yazmak Writing a response 

w.Header().Set("Content-Type", "text/plain")
w.WriteHeader(http.StatusCreated)   // 201
fmt.Fprintln(w, "oluşturuldu")

// Sıra önemli: header → status → body

Önce header'lar, sonra status, en son body — bu sıra. Headers first, status next, body last — this order matters.

Status hiç yazmazsan Go ilk yazıdan önce 200 OK gönderir. http.Status... sabitleri okunaklı: StatusOK, StatusNotFound. If you skip the status, Go sends 200 OK before the first write. Use http.Status... constants — they're self-documenting.

JSON döndürmek? Returning JSON?

7/11

📦 JSON döndür Returning JSON 

type Kitap struct {
    Baslik string `json:"baslik"`
    Yazar  string `json:"yazar"`
}

func handler(w http.ResponseWriter, r *http.Request) {
    k := Kitap{"Tutunamayanlar", "Atay"}
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(k)
}

json.NewEncoder(w).Encode(x) = struct → JSON → cevap. json.NewEncoder(w).Encode(x) = struct → JSON → response.

Backtick'lerdeki json:"..." alan adlarını JSON'a eşler. Stdlib hızlı ve güvenli — başka kütüphane lazım değil. The backtick json:"..." tags map field names to JSON. Stdlib is fast and safe — no third-party needed.

func kitapEkle(w http.ResponseWriter, r *http.Request) {
    var k Kitap
    err := json.NewDecoder(r.Body).Decode(&k)
    if err != nil {
        http.Error(w, "geçersiz JSON", http.StatusBadRequest)
        return
    }

    // k artık doldu, kullanabilirsin
}

json.NewDecoder(r.Body).Decode(&x) = body → struct. json.NewDecoder(r.Body).Decode(&x) = body → struct.

http.Error(w, mesaj, kod) = "kötü istek" cevabını tek satırda gönderir. Hata kontrolünü atlama. http.Error(w, msg, code) sends a clean error response in one line. Don't skip error checks.

Tüm handler'lara ortak iş? Common work for all handlers?

🧅 Middleware Middleware 

func logging(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("%s %s", r.Method, r.URL.Path)
        next.ServeHTTP(w, r)
    })
}

http.ListenAndServe(":8080", logging(mux))

Middleware = "handler'ı sarmalayan handler". Middleware = "a handler that wraps another handler".

Log, auth, CORS, rate limit — hepsini middleware ile ekle. Üst üste sarmalanır: auth(logging(mux)). Logging, auth, CORS, rate limit — add them as middleware. Stackable: auth(logging(mux)).

Programı düzgün kapatmak? Clean shutdown?

10/11

🛑 Graceful shutdown Graceful shutdown 

srv := &http.Server{Addr: ":8080", Handler: mux}

go srv.ListenAndServe()

// Ctrl+C bekle
sig := make(chan os.Signal, 1)
signal.Notify(sig, os.Interrupt)
<-sig

ctx, _ := context.WithTimeout(context.Background(), 5*time.Second)
srv.Shutdown(ctx)    // devam eden istekler bitsin

Sunucu kapanırken devam eden istekleri bitirmesini bekle. When shutting down, let in-flight requests finish.

Production için zorunlu kalıp. Shutdown yeni isteği reddeder, eskileri bitirir, sonra döner. Mandatory in production. Shutdown rejects new requests, drains old ones, then returns.

📍 Niye pointer? Why pointers?

📦vs📍

değer = kopya · pointer = adres value = a copy · pointer = an address

Pointer = bir değerin nerede tutulduğunun adresi. A pointer is the address where a value lives.

İki amaç: (1) function içinden dışarıdaki değeri değiştirmek, (2) büyük yapıları kopyalamadan paylaşmak. Two reasons: (1) mutate outside data from inside a function, (2) share large structs without copying.

Adresi nasıl alırsın? How do you get an address?

2/10

🆔 & ile adres al Take an address with & 

yas := 12
p := &yas         // p, yas'ın adresini tutar

fmt.Println(yas)   // 12
fmt.Println(p)     // 0xc0000180a8 gibi

& bir değişkenin adresini verir. & takes a variable's address.

Adres bir sayı (bellek konumu). Direkt yazdırırsan tuhaf görünür ama önemli değil — pointer'ı kullanırsın, yazdırmazsın. An address is a number (memory location). It looks ugly when printed but doesn't matter — you use pointers, you rarely print them.

Adresteki değere ulaşmak? Reading the value at the address?

3/10

👁️ * ile değere bak Read the value with * 

yas := 12
p := &yas

fmt.Println(*p)    // 12 — adresteki değer

*p = 13            // adresteki değeri değiştir
fmt.Println(yas)   // 13 — yas da değişti!

*p = "p'nin işaret ettiği değer". *p = "the value p points to".

Buna dereference denir. & adres alır, * adresten değer çeker. Birbirinin tersi. This is called dereferencing. & takes an address, * reads through one. Inverses of each other.

Pointer'ın tipi? A pointer's type?

4/10

📦 Pointer tipi: *T Pointer type: *T 

var p *int            // int'e pointer
var kp *Kitap         // Kitap'a pointer

yas := 12
p = &yas               // p artık yas'ı işaret ediyor

*Tip = "şu tipe işaret eden pointer". *Type = "a pointer to that type".

int bir sayı tutar. *int bir sayının adresini tutar. Farklı tipler. int holds a number. *int holds the address of one. Different types.

Boş pointer? Empty pointer?

5/10

❓ nil — hiçbir yere işaret etmiyor nil — points to nothing 

var p *int          // p == nil

if p == nil {
    fmt.Println("henüz adres yok")
}

Pointer'ın sıfır değeri nil'dir. A pointer's zero value is nil.

"Optional" tipi gibi düşün — değer olabilir, olmayabilir. nil kontrolü Go kodunda çok yaygındır. Think of it like an "optional" — value may or may not be there. nil checks are very common in Go.

Function'a pointer geçmek? Passing a pointer to a function?

6/10

🛠️ Function ile değiştirmek Mutating through a function 

// Değer alır → kopya, dışarı yansımaz
func artir(n int)   { n++ }

// Pointer alır → asıl değişkeni değiştirir
func artir(n *int)  { *n++ }

yas := 12
artir(&yas)
fmt.Println(yas)   // 13

Function dışarıyı değiştirsin istiyorsan, pointer geç. If a function should change outer data, pass a pointer.

Go her zaman kopya ile çalışır. Pointer kopyalansa da işaret ettiği değer aynı kalır — bu yüzden değişiklik dışarıdan görülür. Go always copies. Pointers get copied too — but they point to the same value, so changes are visible outside.

Struct ile en sık kullanım? The most common use with structs?

7/10

📋 Struct'a pointer Pointer to a struct 

k := Kitap{Baslik: "X", Yil: 2020}
kp := &k

kp.Yil = 2021          // otomatik dereference!
// Aynı şey: (*kp).Yil = 2021

Struct pointer'ında nokta otomatik dereference yapar. With struct pointers, the dot dereferences automatically.

kp.Yil daha okunaklı, (*kp).Yil ile aynıdır. Go bu iyiliği bize yapar. kp.Yil reads better and is identical to (*kp).Yil. Go does this for you.

nil pointer'a dokunmak? Touching a nil pointer?

8/10

💥 nil dereference = panic nil dereference = panic 

var p *Kitap      // nil

fmt.Println(p.Baslik)
// 💥 panic: runtime error:
//    invalid memory address

nil pointer'ın değerine bakmak panic verir. Reading through a nil pointer panics.

Şüphedeysen önce kontrol et: if p != nil { ... }. Çoğu Go bug'ı budur. When in doubt, check first: if p != nil { ... }. Most Go bugs are this.

Slice/map'e pointer gerekir mi? Do slices/maps need pointers?

9/10

🪞 Slice/map zaten "referans" Slices/maps are already "reference-like" 

// Hatalı — gereksiz pointer:
func ekle(s *[]int, x int) { *s = append(*s, x) }

// Doğru — slice header zaten pointer içerir:
func ekle(s []int, x int) []int {
    return append(s, x)
}

Slice ve map'lere ayrıca pointer alma. İçeriden zaten paylaşılır. Don't take pointers to slices or maps. They share data already.

Onlar zaten içlerinde bir pointer barındırır. *[]int görürsen genelde yanlıştır. They contain a pointer internally. Seeing *[]int is usually a mistake.

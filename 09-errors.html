🚨 Hata bir değerdir Errors are values 

// Diğer dillerde:
try {
    parse(input)
} catch (e) { ... }

// Go'da:
sonuc, err := parse(input)
if err != nil { ... }

Go'da hata "fırlatılmaz" — döndürülür. In Go, errors aren't thrown — they're returned.

Try/catch yok. Hata da diğer dönüş değerleri gibi sıradan bir değer. Açık, görünür, görmezden gelinemez. No try/catch. An error is just another return value. Visible, explicit, hard to ignore.

"Hata" tipi nedir? What is the "error" type?

2/11

📜 error interface'i The error interface 

type error interface {
    Error() string
}

Tek metot: Error() string. Karşılayan her tip bir error'dur. One method: Error() string. Anything that has it is an error.

Stdlib'in en küçük ve en güçlü interface'i. Kendi hatanı yazmak da kolay — Bölüm 8'i hatırla. Stdlib's smallest, mightiest interface. Writing your own is easy — recall Chapter 8.

Hata nasıl döndürülür? How do you return one?

3/11

📤 Hata döndür Return an error 

func bol(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("sıfıra bölme")
    }
    return a / b, nil
}

Standart imza: (sonuç, error). Hata yoksa nil. Standard signature: (result, error). nil means no error.

error her zaman dönüş listesinin son elemanıdır. Konvansiyon — ihlal etme. error is always the last return value. Convention — don't break it.

Karşı tarafta nasıl kontrol edilir? How does the caller check?

4/11

✅ if err != nil — Go ritüeli if err != nil — Go's ritual 

sonuc, err := bol(10, 0)
if err != nil {
    return err           // veya log et, kullanıcıya göster, vs.
}

// Buraya ulaştıysak: hata yok, sonuc güvenli
fmt.Println(sonuc)

Her hata dönen çağrıdan sonra hemen kontrol et. After every call returning an error: check it immediately.

Bu kalıbı çok yazacaksın. Geveze görünebilir ama bu Go'nun gücü — gizli sürpriz yok. You'll write this a lot. Looks verbose, but it's Go's strength — no hidden surprises.

Hata yaratmak? Creating an error?

5/11

🆕 errors.New — basit hata errors.New — simple error 

import "errors"

err := errors.New("dosya bulunamadı")

fmt.Println(err)            // dosya bulunamadı
fmt.Println(err.Error())    // dosya bulunamadı

errors.New(metin) = sadece bir metin taşıyan hata. errors.New(text) = an error that carries just a message.

Mesaj küçük harfle başlasın, nokta ile bitmesin. Go konvansiyonu: "dosya yok" ✓ · "Dosya yok." ✗ Lowercase, no trailing period. Go convention: "file not found" ✓ · "File not found." ✗

Mesaja değer eklemek? Adding values to the message?

6/11

🎨 fmt.Errorf — formatlı fmt.Errorf — formatted 

dosya := "config.json"

err := fmt.Errorf("%s okunamadı", dosya)

// → "config.json okunamadı"

fmt.Errorf = Printf ama bir hata döner. fmt.Errorf = Printf, but it returns an error.

Bağlam bilgisi eklemek için ideal: hangi dosya, hangi kullanıcı, hangi adım. Perfect for adding context: which file, which user, which step.

Hata zincirlemek? Chaining errors?

7/11

🪆 %w — hatayı sarmala %w — wrap the error 

_, err := os.Open("x.json")
if err != nil {
    return fmt.Errorf(
        "config yüklenemedi: %w", err,
    )
}

%w alttaki hatayı korur, üstüne bağlam ekler. %w preserves the underlying error and adds context.

Sonuç bir "hata zinciri" olur. Üst katmanlar üst mesajı görür, ama altındaki hata da kaybolmaz — errors.Is/As ile bulunur. You get an "error chain". Upper layers see the top message, but the inner error is preserved — found with errors.Is/As.

Belirli bir hata mı? Is it a specific error?

8/11

🔍 errors.Is — belirli mi? errors.Is — exact match? 

if errors.Is(err, os.ErrNotExist) {
    fmt.Println("dosya yok, varsayılanı kullan")
}

errors.Is hata zinciri boyunca o değeri arar. errors.Is walks the error chain looking for that value.

String karşılaştırma yapma! err.Error() == "..." kırılgandır. Sentinel değerleri için errors.Is. Don't compare strings! err.Error() == "..." is fragile. Use errors.Is for sentinel values.

Belirli bir tipte mi? Is it a specific type?

9/11

🔎 errors.As — bu tip mi? errors.As — this type? 

var pathErr *os.PathError

if errors.As(err, &pathErr) {
    fmt.Println("dosya:", pathErr.Path)
    fmt.Println("işlem:", pathErr.Op)
}

errors.As hata zincirinde o tipte bir hata arar. errors.As hunts the chain for an error of that type.

Bulursa hedef pointer'ı doldurur ve true döner. Yapısal hatalardan veri okumanın güvenli yolu. If found, fills the target pointer and returns true. Safe way to read data from typed errors.

Kendi hata tipim? My own error type?

10/11

🛠️ Özel hata tipi Custom error type 

type YasHatasi struct {
    Yas int
}

func (e *YasHatasi) Error() string {
    return fmt.Sprintf("geçersiz yaş: %d", e.Yas)
}

err := &YasHatasi{Yas: -1}

Error() string metodu olan her tip bir error'dur. Any type with an Error() string method is an error.

Hatalara veri ekle (yaş, satır numarası, dosya adı). errors.As sayesinde bu veriye sonradan erişilir. Attach data (age, line number, filename). errors.As lets callers reach that data later.

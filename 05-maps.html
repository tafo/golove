🗺️ Niye map? Why a map?

🔑→📦

"İstanbul" → 16.000.000
"Ada" → 12 yaşında "Istanbul" → 16,000,000
"Ada" → age 12

Anahtar verirsin, değer alırsın. Give a key, get a value.

Slice index'le bulur (0, 1, 2…). Map istediğin anahtarla bulur (string, int, vb). Telefon rehberi gibi. Slices use indexes (0, 1, 2…). Maps use any key (string, int, etc). Like a phone book.

Nasıl yaratırız? How do we create one?

2/11

🔑 Map yarat Create a map 

// 1. Doğrudan değerlerle
yaslar := map[string]int{
    "Ada":    12,
    "Mehmet": 25,
}

// 2. Boş başlat
nufus := make(map[string]int)

map[anahtarTipi]değerTipi — iki yoldan biri. map[keyType]valueType — pick one of two ways.

Literal yazım hazır verilerle başlar. make boş bir map verir, sonradan doldurursun. Literal form starts with values. make gives an empty map you fill later.

Değer almak? Reading a value?

3/11

📖 Değer oku Read a value 

yaslar := map[string]int{
    "Ada": 12,
}

yas := yaslar["Ada"]      // 12
yok := yaslar["Zeynep"]   // 0  (sıfır değer!)

Anahtar yoksa sıfır değer döner — hata değil. Missing key returns the zero value — not an error.

Bu Go'nun klasik tuzağı. yaslar["Zeynep"] → 0 sana "var ve 0" mı diyor, "yok" mu diyor? Sonraki kart açıklıyor. A classic Go trap. Does yaslar["Zeynep"] → 0 mean "exists but 0" or "missing"? Next card clarifies.

Var mı yok mu? Is it there or not?

4/11

🔍 Var mı? — comma ok Is it there? — comma ok 

yas, varMi := yaslar["Ada"]
if varMi {
    fmt.Println(yas)        // 12
} else {
    fmt.Println("yok")
}

İkinci dönüş — bool — "anahtar var mıydı?" sorusunu cevaplar. Second return — a bool — answers "did the key exist?".

Buna "comma ok" deyimi denir. Çoğu zaman value, ok := şeklinde yazılır. Map'lere has bir kalıp. This is called the "comma ok" idiom. Usually written value, ok :=. A pattern unique to maps.

Yeni bir eşleme eklemek? Adding a new entry?

5/11

✏️ Yaz veya güncelle Write or update 

yaslar["Zeynep"] = 30     // yeni eklendi
yaslar["Ada"] = 13        // güncellendi

// Aynı operatör. Yeni mi eski mi map otomatik anlar.

Yazmak ve güncellemek aynı operatör: m[key] = val. Writing and updating use the same operator: m[key] = val.

Anahtar zaten varsa değer değişir. Yoksa yeni eşleme eklenir. Tek satır iki iş. If the key exists, the value is replaced. Otherwise a new entry is added. One line, two jobs.

Silmek? Removing?

6/11

🗑️ delete — anahtarı kaldır delete — remove a key 

delete(yaslar, "Ada")

// "Ada" artık yok
// var olmayan anahtarı silmek hata vermez

delete(map, anahtar) — basit ve sessiz. delete(map, key) — simple and quiet.

Olmayan bir anahtarı silmeye çalışsan da hata vermez. Kontrol etmen gerekmez. Deleting a missing key is a no-op — no error. You don't need to check first.

Hepsini gezmek? Iterating all?

7/11

🔁 for range ile gez Iterate with for range 

for isim, yas := range yaslar {
    fmt.Printf("%s: %d\n", isim, yas)
}

// Sadece anahtarlar:
for isim := range yaslar { ... }

// Sadece değerler:
for _, yas := range yaslar { ... }

for k, v := range m — iki şey birden alır. for k, v := range m — gives you both.

Tek değer istiyorsan ikinciyi _ ile boşa at. Map'te 2 dönüş, slice'ta da öyle. Want just one? Discard the other with _. Same for slices.

Sıralı mı geliyor? In order?

8/11

🎲 Sıra rastgele Order is random 

// Aynı kodu 2 kez çalıştır:

// 1. çalışma:  Ada, Mehmet, Zeynep
// 2. çalışma:  Mehmet, Zeynep, Ada
// 3. çalışma:  Zeynep, Ada, Mehmet

// 😱 Her seferinde farklı!

Map'in sırası yok. Her gezişte farklı sırayla gelir. Maps have no order. Iteration order changes every time.

Sıralı çıktı istiyorsan: anahtarları slice'a topla, slices.Sort ile sırala, sonra map'e bak. Want sorted output? Collect keys into a slice, slices.Sort them, then read from the map.

Bir tehlike daha… One more danger…

9/11

⚠️ nil map'e yazma Don't write to a nil map 

var m map[string]int     // nil — değer yok

m["Ada"] = 12
// 💥 panic: assignment to entry in nil map

// Doğru başlatma:
m = make(map[string]int)
m["Ada"] = 12     // ✓

Sadece var m map[...] tanımladıysan map nil'dir. Yazamazsın. If you only write var m map[...] the map is nil. You can't write to it.

Okumak güvenli (sıfır değer döner). Yazmak panic. make veya literal ile başlat. Reading is safe (zero value returns). Writing panics. Initialize with make or a literal.

Boyut? Size?

10/11

📊 len — kaç eşleme var len — how many entries 

yaslar := map[string]int{
    "Ada": 12,
    "Mehmet": 25,
}

len(yaslar)   // 2

len(m) — eşleme sayısı. len(m) — number of entries.

Aynı len, slice'larda da çalışıyordu. Go'nun ortak fonksiyonları azdır — bir kez öğren, her yerde kullan. Same len works on slices too. Go's built-ins are few — learn once, use everywhere.

Tek cümlede özet? One-line summary?

11/11

🎯 Özet Summary

map = anahtar → değer

map = key → value

Yarat → yaz/oku → var mı sor → sil → gez. Create → write/read → check → delete → iterate.

🔑 yarat: make(map[K]V) veya literal
📖 oku: v := m[k] · yoksa sıfır değer
🔍 var mı: v, ok := m[k]
✏️ yaz: m[k] = v
🗑️ sil: delete(m, k)
🔁 gez: for k, v := range m (sıra rastgele)
⚠️ tuzak: nil map'e yazılmaz 🔑 create: make(map[K]V) or literal
📖 read: v := m[k] · zero value if missing
🔍 exists: v, ok := m[k]
✏️ write: m[k] = v
🗑️ delete: delete(m, k)
🔁 iterate: for k, v := range m (random order)
⚠️ trap: can't write to nil map
